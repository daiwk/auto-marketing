# Quant Trader Lab

一个通过本地网页运行的美股量化研究与纸面交易平台。用户在页面选择实验模式、模型、
时间窗口和策略参数，后台执行回测，页面实时展示每个 Agent 的工作过程、最终收益曲线和
实验指标。

> 本项目仅用于研究和纸面模拟，不构成投资建议，不连接券商，也不能发送真实订单。

## 安装

需要 Python 3.12 或更高版本：

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
```

代码库已经包含 2023—2025 年的本地日线数据，正常使用不需要再次下载行情。

## 启动网站

以后所有配置和实验都从网页完成，只需要启动：

```bash
quant-trader web
```

命令会自动打开浏览器。没有自动打开时，复制终端打印的完整地址。指定端口或禁止自动
打开浏览器：

```bash
quant-trader web --port 8080 --no-open-browser
```

网站只监听本机 `127.0.0.1`，地址包含随机访问令牌。不要把地址转发到公网。

## 在页面运行实验

1. 选择实验模式和执行引擎。
2. 选择开始日期和结束日期。
3. 展开“回测与算法参数”，调整股票池、资金、仓位和交易成本等参数。
4. TradingAgents 模式下设置审核次数和审核机会选择方式。
5. 点击“提交后台运行”。
6. 在“中间过程监控”查看实时日志，在 Agent 看板查看每个角色的结论。
7. 任务完成后查看收益、回撤、成交数和可交互净值曲线。

关闭网页不会中止后台任务。停止启动网站的终端进程会终止仍在运行的子任务。每次运行的
配置和结果保存在 `web-runs/<任务 ID>/`，不会修改默认配置文件。

## 实验模式

- **规则基线**：完全离线的确定性规则策略。
- **TradingAgents 多 Agent**：市场、情绪、新闻、基本面、研究、交易和风险等角色共同
  复核规则候选。
- **FinMem**：使用分层记忆辅助候选决策。
- **QuantaAlpha**：在受限因子 DSL 中生成和筛选候选因子。
- **Alpha Arena**：比较已经完成的 FinMem 或 QuantaAlpha 实验。

LLM 只能维持、减仓或拒绝规则已经选出的候选，不能绕过风险限制，也不能创建规则之外的
股票和订单。因此 LLM 曲线不一定优于规则曲线。

## 模型配置

### 本地 Codex

页面选择“本地 Codex”。网站复用本机 Codex CLI 的登录状态，不需要在页面填写 Key。

### 本地 Trae X

页面选择“本地 Trae X”，再从下拉框选择本机 `traex models` 返回的可用模型。模型选择只
影响当前任务，不修改 Trae X 的全局配置。

### 国内 MiniMax M3

启动网站前在同一个终端设置环境变量：

```bash
export MINIMAX_API_KEY='你的 MiniMax API Key'
quant-trader web
```

默认使用国内地址 `https://api.minimaxi.com/v1` 和 `MiniMax-M3`。API Key 只从启动网站的
终端环境读取，不会显示在页面，也不会写入配置、日志和任务结果。

## 时间窗口与指标预热

页面默认开始日期为 `2023-01-01`，结束日期为 `2026-01-01`。结束日期不包含当天，因此
默认覆盖内置的 2023—2025 年行情。

策略要求至少 252 个交易日历史，并使用 200 日均线、20/60/120 日动量、20 日波动率、
14 日 ATR 和流动性指标。历史不足时策略保持现金，LLM 也不会被调用。页面曲线开头出现
一段水平线通常是指标预热，而不是任务卡住。

## TradingAgents 审核次数

一次“审核机会”是某个周度调仓日中，一个通过规则筛选、准备交给 LLM 复核的候选标的。
日志中的总机会数，例如 341，是整个时间窗口内候选数量的合计，不是模型请求次数。一个
完整 TradingAgents 审核会依次调用多个角色。

页面提供三种选择方式：

- **均匀分布（推荐）**：把有限审核次数分散到整个回测区间。
- **最前 N 次**：只审核最早出现的 N 个机会。
- **指定序号**：填写从 1 开始、逗号分隔的机会序号，数量必须等于审核次数。

例如总共有 341 次机会，设置 4 次均匀审核时，会选择第 1、114、228、341 次。未被选中的
机会使用本地规则回复，不产生模型调用。

## 页面监控与结果

TradingAgents 看板会保留每次完整审核。顶部的“第 1 次、第 2 次……”可以切换不同日期和
标的，查看每个角色的状态、立场、信心、依据、风险、交易员建议和最终组合决策。

最终净值曲线同时展示：

- 规则策略
- LLM 策略
- SPY 买入持有基准

鼠标移动到曲线上可以查看对应日期和各策略净值。LLM 使用虚线；曲线高度重合时，图例会
提示走势近乎重合。

## 常见问题

### 页面一直显示运行中

查看“中间过程监控”。MiniMax、Codex 或 Trae X 调用可能较慢；超时和模型原始错误会直接
显示在日志中。Trae X 模型可能需要排队，可以换一个模型或减少完整审核次数。

### 为什么规则和 LLM 长时间都没有交易

规则首先需要 252 个交易日完成指标预热，之后还必须同时满足价格高于 200 日均线、20/60
日动量为正、流动性达标等条件。没有规则候选时，LLM 没有可审核对象，两条曲线都会保持
现金。

### 为什么 LLM 效果不如规则策略或 SPY

LLM 只是受约束的风险复核层，只能降低规则候选的仓位。少量审核也只是抽样，不能代表整个
回测区间持续使用模型。比较效果时应同时查看审核覆盖率、交易成本、最大回撤和不同时间
窗口，不能只看单次总收益。

### 数据或日期报错

所选时间窗口必须位于本地行情数据覆盖范围内，且开始日期早于结束日期。默认数据覆盖
2023—2025 年。
